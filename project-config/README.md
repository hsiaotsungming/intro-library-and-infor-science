# Project Config

This folder stores working knowledge for building the website.

It is separate from `site/` because these files describe how the site should be made; they are not themselves publishable website content.

## Structure

- `style/`: writing style, visual style, UI tone, and content presentation rules.
- `technical/`: implementation choices, GitHub Pages deployment notes, and site architecture decisions.
- `workflow/`: collaboration rules, publishing rules, and unit-level work process.
- `skills/`: reusable instructions for Codex or future collaborators, including project logging, publishing, style consistency, page asset rules, and display image cropping.

## Rule

Keep these files practical and durable. Update them when a decision affects future work.
